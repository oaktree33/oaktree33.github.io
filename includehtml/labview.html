In junior year, I had to "write" many LabView programs for the Digital Electronic class and lab. Some of them include integrated circuit chip emulation, serial communication through fiber optic cable using DAQ, and motor speed control.
However, being someone who had previous background in programming (in python and java), I found graphical programming time-consuming and limited. Although it is great at quickly whipping up decent graphical UI, it did not seem to be geared for serious programming.
Due to these facts I decided to research "text based coding in LabView" and present it to the class.

One might use text-based codes in LabView because it....:

  • saves space on the Block Diagram
  • is faster to type than drag and drop certain features
  • allows to take advantage of pre-existing text-based codes
  • extends the functionality of LabView
  • exercises both hands instead of just one...

For the LabView 2011, there were four different methods in which to use text-based coding:

  • Formula Node
  • Mathscript
  • Matlab node
  • DLL (C/C++)

I decided to take a simple and comical program as an example. The program was simulating the projection of an object thrown upward at a user given velocity, where the object was an image of a cat.

I implemented the same program 4 times, using the aforementioned text-based coding methods within LabView.

Formula Node

There is a graphical element within LabView called the "formula node", placing this element onto the block diagram will take a form of a text-box that can be connected to input and output elements.

Matlab Node

There is a programming language platform called Matlab that is frequently used for complex calculations and mathematical analysis. If you have access to Matlab, then you can use the matlab node in labview to do computations through matlab.
Below is the image of the same cat jump program implemented using the matlab script node.

As you can see, the setup is very similar to mathscript and the formula node. The matlab node will send the typed script to the matlab server running locally on the machine, and then the server will run the script and return the result to the Labview.

DLL (C/C++)

LabView has an element called the "call library function node". This node allows you to use a function/method that is defined in a DLL file written using C or C++. Each node can be used for one function within the DLL.
